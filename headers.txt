Request Headers
1. User-Agent: PostmanRuntime/7.43.0 -- This indicates the client making the request.
2. Accept: */* -- This indicates that the client can accept any response type.
3. Cache-Control: no-cache -- This instructs the server not to cache the request and always fetch fresh data.
4. Postman_Token: <unique-value> -- This is a unique identifier generated by Postman for each request to prevent request duplication.
5. Host: www.googleapis.com –  This specifies the server’s domain where the request is being sent. 
6. Accept-Encoding: gzip, deflate, br – This specifies the encoding methods that the client supports for compression.
7. Connection: keep-alive – This keeps the network connection open for reuse.

Response Headers
1. Content-Type: application/json; charset=UTF-8 – This specifies that the response body is in JSON format and uses UTF-8 encoding. 
2. Vary: Origin, X-Origin, Referer – This are instructions for caching servers to differentiate content based on Origin, X-Origin, or Referer.
3. Content-Encoding: gzip – This indicated that the response 9is compressed using gzip reduce data size. 
4. Date -- The date and time when the response was generated.
5. Server: ESF – This identifies the server that processed the request.
6. X-XSS-Protection: 0 – This disables Cross-Site Scripting protection.
7. X-Frame-Options: SAMEORIGIN – This prevents the response from being embedded in a frame on another domain for security.
8. X-Content-Type-Options: nosniff – This instructs the browser not to guess the MIME type and only use the declared content type.
9. Alt-Svc: h3=":443"; ma=2592000,h3-29=":443"; ma=2592000 – This indicates alternative services or protocols that can be used to improved performance. 
10. Transfer-Encoding: chunked – This specifies that the response is sent in chunks instead of a single block.
